My Projects:

Shark Autonomous Robot Navigation System

While at Shark, I had the opportunity to lead the lidar-based robot navigation subsystems for the RV2000 and RV2500 series robots. I was responsible for evaluating main navigation sensors to choose the right one for the job based on things like performance, reliability, vendor relationship and price point. Once we had a sensor selected I was mainly responsible for progressing the subsystem through iterative changes driven by user feedback, a suite of tests which benchmarked performance and issues uncovered during in home testing. This included defining and managing design qualification test protocols, ensuring that we don't regress from build to build. For the tests, I designed electromechanical fixtures to measure things like sensor accuracy or operation under extreme temperatures. I worked directly with consumer data and returned robots to identify overarching problems that needed attention and suggested solutions. I also gave demonstrations to key stakeholders and executives in the company, displaying the technology through demonstrations and fielding any questions about the system.

Skills: RCCA, Sensor Selection, Data Analysis, User Interaction, Problem Solving, Test Development, Electromechanical Fixture Fabrication, Presentation

HALT Testing

While at Shark, I lead a cross-functional team of engineers to select the lidar sensor used in our navigation system. We developed and executed accelerated life tests on a selection of lidar sensors to predict performance in our advanced navigation robots. I modified basic lidar software tools provided by software developers to optimize for the testing and provide an easy to use interface for the test engineers. After the testing, I worked with our systems architect to analyze the collected data and compile it into easy to understand graphics. Finally I presented the findings to executives and the wider team to make the decision of which sensor to select.

Skills: Test development, Data analysis, Data Visualization, Collaboration, Presentation

Autonomous Mapping with TurtleBots

Using the turtlebot 2 platform, I wrote code to autonomously explore and map an area in real-time. I implemented a frontier based exploration algorithm using A* with a heavily modified heuristic built to explore an area as efficiently as possible while avoiding obstacles. This project was my first experience using ROS, allowing me to pick up on fundamentals and build on my previous experience in Linux.

Skills: Algorithms, Path Planning, ROS, Linux, Collaboration

3D Printing

After working with the original Makerbots during a summer job just before college, I caught the 3D printing bug and purchased my own printer. After gaining some experience, I worked in the WPI robotics lab to help maintain the heavily utilized Taz printers which were available to robotics majors. I’ve had a few 3D printers, with my most recent being an Ender 3, which I constantly improve through firmware and hardware modifications. I love to push the limit of both the platform's capabilities and my own skills and creativity. My most recent upgrade to my Ender 3 involves mesh bed leveling which is automated through the use of a hall effect driven probe. Next, I am looking at upgrading the stepper motor drivers and fans to make the machine quieter to run. Exploring the 3D printing universe is always exciting as having access to the base firmware of the machine alongside lots of open source firmwares always leaves me feeling like I can do something to improve print quality and ensure print success.

Skills: 3D Printing, Problem solving, C/C++ Programming, Self-teaching
